## Brief overview
Guidelines for using plain ASCII characters in prose content (commit messages, PR text, documentation, comments, user-facing strings).

## Character substitutions
- Use `'` and `"` instead of smart/curly quotes (`'`, `"`, `"`)
- Use `-` instead of en-dash (`–`) or em-dash (`—`)
- Use `...` instead of ellipsis (`…`)
- Avoid decorative Unicode like `•`, `→`, or non-breaking spaces

## When exceptions apply
- Technical formats that require specific Unicode characters
- User explicitly requests Unicode characters
- Code identifiers or strings where Unicode is intentional
